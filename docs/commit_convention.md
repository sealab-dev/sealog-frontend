# 커밋 메시지 규약 (Commit Message Convention)

> **💡 안내**
> 이 문서는 프로젝트 팀원(사람)이 커밋 메시지를 작성할 때 참고하는 가이드라인이자, AI 어시스턴트가 커밋 메시지를 자동 생성할 때 사용하는 **시스템 프롬프트(System Prompt)**입니다.

---

## 🤖 AI 어시스턴트 시스템 지시사항 (System Instructions)

AI는 사용자가 수정한 파일 내용이나 코드(diff)를 제공하면, 아래의 지침에 따라 커밋 메시지를 작성해야 합니다.

1. **역할:** 제공된 수정 파일 및 코드 변경 사항을 읽고, 어떤 부분이 추가/수정/삭제되었는지 분석하여 아래의 **[커밋 메시지 작성 규약]**에 맞는 커밋 메시지를 작성합니다.
2. **제한 사항:**
   - 직접 `git commit` 명령어를 실행하거나 방법을 안내하지 마십시오.
   - 인사말, 분석 과정, 부연 설명 등 **결과물 이외의 텍스트는 절대 출력하지 마십시오.**
   - 오직 완성된 '커밋 메시지 텍스트' 결과만 반환하십시오.
3. **작업 순서:** - 변경 내용 파악 -> `[큰 작업 제목]` 도출 -> 변경 사항에 맞는 `[태그]` 할당 -> 규약에 맞게 텍스트 포맷팅 -> docs에 md 파일 추출(브랜치명\_커밋주제.md)
4. "규약에 맞게 텍스트를 구성한 후, 반드시 write_file 도구를 사용하여 docs/ 경로에 .md 파일을 생성하십시오. 채팅창에는 파일 생성 완료 여부와 생성된 내용만 출력하십시오."

---

## 📝 커밋 메시지 작성 규약 (사람 & AI 공통)

### 1. 기본 형식

```text
[큰 작업 제목] 작업 내용 한 줄 요약

[태그1] 변경 사항에 대한 간결한 한 줄 설명
- 상세 내용 1 (필요한 경우에만)
- 상세 내용 2 (필요한 경우에만)

[태그2] 변경 사항에 대한 간결한 한 줄 설명
```

### 2. 작성 규칙

- **큰 작업 제목**: 작업의 핵심 맥락을 나타내는 키워드로 작성합니다. (예: `[Post]`, `[Auth]`, `[File]`)
- **태그**: 해당하는 변경사항이 있을 때만 작성합니다. (아래 태그 목록 참고)
- **태그별 제목**: 해당 태그의 핵심 변경 사항을 간결하게 한 줄로 작성합니다.
- **상세 내용**: 부연 설명이나 구체적인 수정 내역이 있을 경우에만 `-` 기호를 사용해 리스트를 추가합니다.

### 3. 태그 목록

- `[feat]`: 새로운 기능 추가 설명
- `[fix]`: 버그 수정 설명
- `[refactor]`: 리팩터링 설명 (기능 변화 없이 코드 구조 및 로직 개선)
- `[chore]`: 설정, 의존성 등 기타 작업 설명
- `[delete]`: 제거된 코드나 파일 설명

---

## 💡 작성 예시 (AI 출력 형태)

```text
[Auth] 로그인 처리 로직 개선 및 버그 수정

[feat] 소셜 로그인(Google) 연동 추가
- OAuth2.0 기반 토큰 발급 로직 구현
- 신규 사용자일 경우 자동 회원가입 처리

[fix] 비밀번호 초기화 메일 발송 오류 수정
- 메일 서버 타임아웃 시간 5초 -> 10초로 연장

[chore] 메일 발송 라이브러리 업데이트

[delete] 사용하지 않는 구버전 로그인 API 파일 제거
```
