서버 상태란 무엇인가

React 애플리케이션에서 상태는 크게 두 가지로 나뉩니다. 클라이언트 상태와 서버 상태입니다. 클라이언트 상태는 UI의 열림/닫힘, 선택된 탭 같은 로컬 데이터를 의미하지만, 서버 상태는 API를 통해 가져오는 원격 데이터를 의미합니다.

서버 상태는 다음과 같은 고유한 특성을 가집니다.

  • 당신이 소유하지 않는 원격 위치에 저장됩니다
  • 비동기 API를 통해서만 접근 및 수정할 수 있습니다
  • 다른 사람에 의해 변경될 수 있어 "오래된(stale)" 상태가 될 수 있습니다
이미지 영역

TanStack Query 기본 설정

TanStack Query를 프로젝트에 세팅하는 것은 간단합니다. QueryClient를 생성하고 QueryClientProvider로 앱을 감싸주면 됩니다. 

tsximport { QueryClient, QueryClientProvider } from '@tanstack/react-query'

const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: 1000 * 60 * 5,
      gcTime: 1000 * 60 * 10,
    },
  },
})

export function App() {
  return (
    <QueryClientProvider client={queryClient}>
      <Router />
    </QueryClientProvider>
  )
}
staleTime과 gcTime은 헷갈리기 쉽습니다. staleTime은 데이터가 "신선"하다고 간주되는 시간이고, gcTime은 캐시에서 제거되기까지의 시간입니다.

useQuery 핵심 패턴

useQuery는 가장 기본적인 훅으로, 데이터를 fetching하고 캐싱합니다. 쿼리 키는 배열로 구성하며, 이 키를 기준으로 캐시가 관리됩니다. 

tsxfunction useUserPosts(userId: string) {
  return useQuery({
    queryKey: ['posts', userId],
    queryFn: () => fetchUserPosts(userId),
    enabled: !!userId,
    select: (data) => data.filter(post => !post.hidden),
  })
}

쿼리 키 설계 전략

쿼리 키는 단순한 식별자가 아니라 캐시 전략의 핵심입니다. 계층적으로 구성하면 관련 캐시를 한 번에 무효화할 수 있습니다. 

tsexport const postKeys = {
  all: ['posts'] as const,
  lists: () => [...postKeys.all, 'list'] as const,
  list: (filters: string) => [...postKeys.lists(), { filters }] as const,
  details: () => [...postKeys.all, 'detail'] as const,
  detail: (id: string) => [...postKeys.details(), id] as const,
}

낙관적 업데이트

낙관적 업데이트는 서버 응답을 기다리지 않고 UI를 즉시 업데이트하는 패턴입니다. 사용자에게 빠른 피드백을 줄 수 있지만, 서버 오류 시 롤백 처리가 필요합니다. 

tsxconst mutation = useMutation({
  mutationFn: updatePost,
  onMutate: async (newPost) => {
    await queryClient.cancelQueries({ queryKey: postKeys.detail(newPost.id) })
    const previousPost = queryClient.getQueryData(postKeys.detail(newPost.id))
    queryClient.setQueryData(postKeys.detail(newPost.id), newPost)
    return { previousPost }
  },
  onError: (err, newPost, context) => {
    queryClient.setQueryData(postKeys.detail(newPost.id), context?.previousPost)
  },
  onSettled: (data, error, variables) => {
    queryClient.invalidateQueries({ queryKey: postKeys.detail(variables.id) })
  },
})

무한 스크롤 구현

useInfiniteQuery를 사용하면 페이지네이션 또는 무한 스크롤을 선언적으로 구현할 수 있습니다. 

tsxconst { data, fetchNextPage, hasNextPage } = useInfiniteQuery({
  queryKey: ['posts', 'infinite'],
  queryFn: ({ pageParam = 1 }) => fetchPosts({ page: pageParam }),
  getNextPageParam: (lastPage) => lastPage.nextPage ?? undefined,
  initialPageParam: 1,
})
